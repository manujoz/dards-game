# X01## Visión general

X01 es el modo clásico de "cuenta atrás": cada jugador empieza con una puntuación (301/501/701/901) y debe llegar a **0 exacto**.

## Configuración

- **Puntuación inicial**: 301, 501 (por defecto), 701, 901.
- **In Mode**:
    - **Straight-In** (por defecto): se puede empezar a descontar con cualquier impacto.
    - **Double-In**: no se descuenta hasta acertar un **doble**.
- **Out Mode**:
    - **Straight-Out**: se puede cerrar con cualquier impacto que deje el marcador en 0.
    - **Double-Out** (por defecto): el dardo final debe ser un **doble**.
    - **Master-Out**: el dardo final debe ser **doble o triple**.

## Puntuación

- **Inputs**: segmento (1–20, 25) y multiplicador (1, 2, 3).
- **Cálculo**: puntos = segmento × multiplicador.
- **Actualización**: marcador nuevo = marcador anterior − puntos.

## Condiciones de victoria

Gana el primer jugador que llegue a **0** respetando el **Out Mode**:

- **Double-Out**: el último dardo debe ser un **doble** (incluye **D-BULL**: 25×2 = 50).
- **Master-Out**: el último dardo debe ser **doble o triple**.

## Reglas de Bust

Un **Bust** ocurre si una tirada deja el marcador en un estado inválido. El turno termina inmediatamente y el marcador vuelve al valor que tenía **al inicio del turno**.

Condiciones:

1. **Marcador < 0**: cualquier impacto que baje de 0.
2. **Marcador = 1 (Double/Master-Out)**: es imposible cerrar desde 1, por tanto es bust.
3. **Marcador = 0 (checkout inválido)**: llegar a 0 sin cumplir el multiplicador requerido del Out Mode.

## Decisiones de precisión

- **Bull en Double-Out**: **Inner Bull** cuenta como doble (D25 = 50) para cerrar; **Outer Bull** (25) no.
- **Reversión en Bust**: el marcador vuelve al estado **antes del primer dardo** del turno, aunque el bust ocurra en el 2º o 3º.

## Escenarios canónicos

### Escenario 1: Victoria estándar (Double-Out)

_Config: 501, Double-Out_

- **Inicio turno**: 40
- **Tirada 1**: S20 → 20
- **Tirada 2**: D10 → 0
- **Resultado**: **WIN**

### Escenario 2: Bust por pasarse (marcador < 0)

_Config: 501, Double-Out_

- **Inicio turno**: 20
- **Tirada 1**: D20 (40) → -20 (**BUST**)
- **Resultado**: el marcador vuelve a **20**

### Escenario 3: Bust por dejar 1 (Double-Out)

_Config: 501, Double-Out_

- **Inicio turno**: 19
- **Tirada 1**: S18 → 1 (**BUST**)
- **Resultado**: el marcador vuelve a **19**

### Escenario 4: Bust por checkout inválido (llegar a 0 sin doble)

_Config: 501, Double-Out_

- **Inicio turno**: 16
- **Tirada 1**: S16 → 0 (**BUST**, no es doble)
- **Resultado**: el marcador vuelve a **16**

### Escenario 5: Ejemplo real del problema (51 = 20 + 20 + 11)

_Config: 501, Double-Out_

- **Inicio turno**: 51
- **Tirada 1**: S20 → 31
- **Tirada 2**: S20 → 11
- **Tirada 3**: S11 → 0 (**BUST**, checkout inválido por no ser doble)
- **Resultado**: el marcador vuelve a **51**

### Escenario 6: Double-In

_Config: 301, Double-In_

- **Inicio turno**: 301
- **Tirada 1**: S20 → 301 (no descuenta)
- **Tirada 2**: D20 → 261
- **Tirada 3**: S20 → 241
